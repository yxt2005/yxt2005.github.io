6.1 问题：npm 运行时报错“因为在此系统上禁止运行脚本”

解决办法：

参考：npm 运行时报错“因为在此系统上禁止运行脚本”解决办法_npm n因为在此系统上禁止运行脚本-CSDN博客

这是因为 Windows 系统的 PowerShell 执行策略 默认禁止执行脚本。你可以通过以下步骤解决：

  1. 打开 PowerShell（以管理员身份）。

  2. 输入以下命令以查看当前的执行策略：

    如果显示的是 Restricted，则表示 PowerShell 阻止了脚本的执行。

  3. 运行以下命令来允许执行脚本：

    • RemoteSigned 允许本地脚本执行，但对于从互联网下载的脚本要求它们必须有一个有效的签名。
    • -Scope CurrentUser 只会改变当前用户的设置，避免修改全局设置。

  4. 输入 Y 确认更改。

  5. 重新运行 npm install，问题应该解决。

6.2 问题：GitHub 配置 SSH 密钥的步骤

参考：Github配置ssh key的步骤（大白话+包含原理解释）_github生成ssh key-CSDN博客

解决步骤：

1. SSH 密钥是什么？
  • SSH 密钥是一对加密的密钥（公钥和私钥），用来验证你和 GitHub 之间的连接。通过使用 SSH 密钥，你不需要每次推送代码时都输入 GitHub 的用户名和密码，它提供了更安全、快捷的认证方式。
2. 生成 SSH 密钥

  1. 打开 PowerShell 或 命令行。

  2. 运行以下命令来生成一个新的 SSH 密钥对：

    • -t rsa 指定使用 RSA 算法生成密钥。
    • -C "youremail@example.com" 为密钥添加一个注释（通常是你的电子邮件）。

  3. 系统会询问你将密钥保存在哪个位置。直接按回车键即可将密钥保存在默认位置：

  4. 接着，系统会询问你是否为密钥设置密码。如果你希望每次使用时都输入密码，可以设置；如果不想每次输入密码，直接按回车即可。

3. 查看公钥内容

  1. 生成 SSH 密钥对后，公钥会保存在文件 id_rsa.pub 中。查看公钥内容：

  2. 将显示的公钥内容复制。

4. 将公钥添加到 GitHub
  1. 登录到 GitHub 网站。
  2. 点击右上角的头像，选择 Settings。
  3. 在左侧菜单中选择 SSH and GPG keys。
  4. 点击 New SSH key。
  5. 在 Title 中输入一个名称（例如：My Laptop），然后将复制的公钥粘贴到 Key 区域。
  6. 点击 Add SSH key 保存。
